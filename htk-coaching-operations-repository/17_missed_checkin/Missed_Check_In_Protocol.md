# 17. Missed Check-In Protocol

## Objective

Prevent clients from drifting silently.

## Missed Check-In Definition

A missed check-in occurs when:

- Check-in deadline passes.
- Client has not submitted required weekly update.
- No approved pause or travel note explains the miss.

## Missed Check-In Workflow

1. Confirm check-in was due.
2. Confirm client did not submit.
3. Send missed check-in message.
4. Record missed check-in in CRM.
5. Set next action.
6. If repeated, escalate accountability level.

## First Missed Check-In Message

Use:

> You missed this week's check-in. I need that submitted so I can coach the plan properly. Get it in today if possible, even if the week was rough. The check-in is how we make the next adjustment.

## Second Consecutive Missed Check-In

Use:

> This is the second missed check-in. At this point I do not have enough information to coach the week properly. What is getting in the way of submitting it?

Coach action:

- Move client to Accountability Level 2 or 3.
- Document pattern.

## Third Missed Check-In In Four Weeks

Use:

> We need to reset the standard. HTK only works if you communicate. If the current plan or check-in rhythm is not realistic, tell me now so we can adjust it. If you want coaching to work, check-ins are non-negotiable.

Coach action:

- Escalate to manager.
- Mark retention risk.
- Follow Non-Compliant Client Protocol if needed.

## Decision Rules

Do not punish:

- Approved travel
- Emergency
- Platform issue
- Client communicated ahead of time

Do intervene:

- No communication
- Repeated missed check-ins
- Vague excuses
- Client expects programming changes without feedback

## CRM Fields To Update

- Check-In_Status
- Missed_Check_In_Count
- Last_Check_In_Date
- Next_Action
- Next_Action_Date
- Accountability_Level
- Notes
