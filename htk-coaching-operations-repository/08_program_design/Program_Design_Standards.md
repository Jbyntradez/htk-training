# 8. Program Design Standards

## Objective

Create performance-focused training that matches the client's goal, starting point, constraints, and readiness while preserving HTK standards.

## Program Design Inputs

Every program must be based on:

- Primary goal
- Training age
- Current capacity
- Schedule
- Equipment
- Mobility needs
- Conditioning needs
- Strength needs
- Recovery constraints
- Injury/scope flags
- Client adherence history

## Required Program Components

Every active HTK program must include, unless a documented reason says otherwise:

1. Strength development
2. Conditioning or endurance work
3. Mobility or movement preparation
4. Athletic or explosive exposure when appropriate
5. Recovery/readiness awareness
6. Progression method
7. Weekly execution target

## Program Build Workflow

1. Confirm goal category.
2. Confirm available training days.
3. Select weekly structure.
4. Choose primary adaptations.
5. Select exercise categories.
6. Set volume and intensity conservatively enough for adherence.
7. Add mobility and preparation work.
8. Add conditioning matched to goal.
9. Add explosive work only when readiness supports it.
10. Define progression rules.
11. Define check-in metrics.
12. Document rationale in CRM.

## Weekly Structure Decision Rules

| Client Situation | Recommended Structure |
|---|---|
| New/deconditioned client | 3-4 training days with simple full-body or upper/lower structure |
| Former athlete returning | 3-5 days with strength, conditioning, mobility, and low-dose power |
| Busy high performer | Minimum effective plan with clear non-negotiable sessions |
| Tactical/first responder | Strength, conditioning, loaded movement, mobility, readiness |
| Athlete in-season | Maintain strength/power, manage fatigue, support sport practice |
| Strength-heavy client | Add conditioning, mobility, movement quality, and athletic exposure |

## Progression Standards

Progress when:

- Client completes prescribed work.
- Technique is acceptable.
- Recovery is adequate.
- Check-in confirms readiness.
- No safety flag is present.

Hold steady when:

- Client barely completes work.
- RPE is consistently high.
- Sleep/recovery is poor.
- Adherence is inconsistent.
- Movement quality is unstable.

Regress when:

- Client cannot complete sessions.
- Pain/safety concern appears.
- Technique breaks down.
- Client misses repeated sessions.
- Load or complexity exceeds readiness.

## Program Change Rules

Do not change a program because:

- Client is bored.
- One session felt hard.
- Coach wants novelty.
- Social media trends suggest it.
- Client asks for random exercises.

Change a program when:

- Goal changes.
- Assessment data changes.
- Adherence pattern requires adjustment.
- Recovery is not supporting the plan.
- Client progresses beyond current stimulus.
- Safety or scope issue requires modification.

## Coach Program Review Checklist

- [ ] Goal is clear.
- [ ] Weekly schedule matches client life.
- [ ] Strength work supports goal.
- [ ] Conditioning supports goal.
- [ ] Mobility supports movement needs.
- [ ] Explosive work is appropriate for readiness.
- [ ] Progression is defined.
- [ ] Check-in metrics are defined.
- [ ] Safety flags are addressed.
- [ ] Program rationale is documented.
