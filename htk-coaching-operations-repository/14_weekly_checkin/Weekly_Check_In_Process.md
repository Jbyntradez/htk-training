# 14. Weekly Check-In Process

## Objective

Use weekly check-ins to monitor execution, progress, recovery, obstacles, and accountability.

## Weekly Check-In Workflow

1. Client submits check-in.
2. Coach reviews adherence.
3. Coach reviews training notes.
4. Coach reviews recovery and readiness.
5. Coach reviews goal progress.
6. Coach identifies blockers.
7. Coach decides: continue, adjust, regress, escalate, or accountability intervention.
8. Coach responds with clear feedback.
9. Coach updates CRM.

## Required Check-In Questions

Client should answer:

- What sessions did you complete?
- What sessions did you miss?
- What felt strong?
- What felt limited?
- What was the biggest obstacle this week?
- How was recovery?
- Any pain, injury, or unusual symptoms?
- What is your focus for next week?

## Coach Review Decision Rules

### Continue Plan

Use when:

- Client completed most work.
- Recovery is acceptable.
- No safety flag exists.
- Progress is stable.

### Adjust Plan

Use when:

- Client completed work but recovery is poor.
- Schedule changed.
- Progress stalls with good adherence.
- Client needs different volume/intensity.

### Regress Plan

Use when:

- Client misses repeated work.
- Sessions are too difficult.
- Technique or readiness is not supporting the plan.
- Client is returning after illness/travel/time off.

### Escalate

Use when:

- Client reports injury/pain concern.
- Client requests medical advice.
- Client is non-compliant for multiple weeks.
- Client reports customer-service issue.
- Client becomes hostile or unsafe.

## Weekly Coach Response Format

1. Acknowledge execution.
2. Name the most important pattern.
3. Give one or two priorities.
4. State any program adjustment.
5. Reinforce next week's standard.

Example:

> You got three sessions in, which keeps the week productive. The pattern is that conditioning is still getting skipped when work runs long. This week, keep the strength work as written and treat the first conditioning session as non-negotiable. If time gets tight, do the shorter version rather than skipping it.

## Check-In Completion Standard

A check-in is complete only when:

- Coach reviewed it.
- Client received feedback.
- Any program change is documented.
- Any missed session pattern is noted.
- Any injury/safety flag is escalated.
- CRM is updated.
